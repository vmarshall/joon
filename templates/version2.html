The v2 version of the API just uses Spectacular, ViewSets and Generic DRF Serializers

Go to the Swagger or Redoc links to get CRUD API access

Go to the Admin screen to check data

Curl Examples:

  • Create Quest: curl -X 'POST' 'http://localhost:8000/api/v2/quests/' -H 'accept: application/json' -H 'Content-Type: application/json' -d '{ "title": "I am a new quest", "description": "This should be new", "repeats": "Monday, Wednesday, Friday" }' -u admin:password
  • Assign Repeats to Quest: curl -X 'PUT' \ 'http://localhost:8000/api/v2/repeats/5/' -H 'accept: application/json' \ -H 'Content-Type: application/json' -d '{ "id": 5, "monday": true, "tuesday": true, "wednesday": true, "thursday": true, "friday": true, "saturday": true, "sunday": true, "quest": 13 }''
  • Get all Quests: curl -X 'GET' 'http://localhost:8000/api/v2/quests/' -H 'accept: application/json' -u admin:password
  • Get Quest #13: curl -X 'GET' 'http://localhost:8000/api/v2/quests/13/' -H 'accept: application/json' -u admin:password
