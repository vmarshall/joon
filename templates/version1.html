Quests API v1

General Quests should always appear in the get_quests response, until it gets completed. On the date that it is completed, the completed field should be True. On dates before the completion date, the completed field should be False.

Repeating Quests should only appear when the date queried is on a day that it is supposed to repeat (see the weekday() function in datetime). If the Quest has been completed that day, completed should be True. If it hasnâ€™t been completed on that day, completed should be False.

Assumptions

The following assumptions were made in the design of the v1 API

  • General Quests that are repeatable can't ever be complete
  • Repeatable Quests will return all quests which are not complete and repeatable
  • Repeatable Quests should never have a completed date ( needs error checking )
  • Completed boolean field can only be marked on General Quests ( needs more checks )
